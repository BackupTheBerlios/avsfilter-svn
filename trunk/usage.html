Check avisynth install properly

Download, compile and install avs2yuv. Check avisynth work with using recommendation from avs2yuv page.

First two filters

Create empty avs script:

LoadPlugin ("pipe_source.dll")
PipeSource()
Copy avsload.exe and pipe_source.dll to wine directory c:\windows\system32 or other directory with wine/windows system dll's. Save script name to other directory, which is accessed by wine applications. Use winepath for check accessibility of wine pathname :
# winepath -w .
# winepath -w /home/user/.wine/drive_c/wine_app/avsload.exe
winepath return windows-like path/filenames for directory/file

Start avidemux2, Menu->Edit->Preferences->External Filters, set checkbox "Load external filters" and select directory with avsfilter.so:

Exit from avidemux2 and restart them from console. Read console output for following lines:

** Registering dynamic filters (/home/fahr/Xfilters) **
Searching for files .so
Found 1 candidated
Trying /home/fahr/Xfilters/avsfilter.so
Terminate class is calling in start
Loaded filter AvsFilter, ver 0.2a, version 1
** Done registering dynamic filters**
If all is ok, load test video file and select any video encoder (ie MPEG-4 ASP (Xvid4)). Select Filters->External filters->AvsFilter, add these filter. Press Browse button and select avsload.exe file:

Press Browse button and select script name:

Press Ok button. AvsFilter has been added to the filter list:

Press Close button and select Output view (instead of Input by default). Source clip should be view. Check play mode, seek on clip to begin/end and so on.

Multifilter check.

Create following script :

LoadPlugin ("pipe_source.dll")
PipeSource().TurnLeft()
Save it to the other script file and load file to new avsfilter. Source clip is rotate to Left side. If all is ok, several avisynth scripts work on your avidemux2 under Linux properly.
